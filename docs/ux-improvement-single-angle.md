# UX 개선: 정면 사진만 필수, 좌/우 선택사항

## 🎯 현재 문제점

1. **3장 강제 촬영**: MultiAngleCameraCapture가 정면/좌/우 3장을 모두 요구
2. **사용자 불편**: 좌/우 촬영이 번거로움
3. **이탈률 증가**: 복잡한 프로세스로 인한 사용자 이탈 가능성

## ✅ 개선 방안

### 1. 정면만 필수, 좌/우 선택사항

#### 백엔드 지원 확인
- ✅ 이미 단일 이미지 지원 (`analyzeVisionLegacy`)
- ✅ 좌/우는 선택적으로 처리 (`if (sideImages.length > 0)`)
- ✅ 정면만으로도 충분한 분석 가능

#### 프론트엔드 개선
1. **기본 플로우**: 정면만 촬영 → 즉시 분석 시작 가능
2. **선택적 개선**: "더 정확한 분석을 위해 좌/우 사진도 촬영하세요" (선택사항)
3. **카메라 모드**: 
   - 기본: 정면만 촬영
   - 고급: 정면 + 좌/우 (선택)

### 2. UX 플로우 개선

#### 옵션 A: 단계별 선택 (권장)
```
1. 정면 사진 촬영/선택
   ↓
2. "분석 시작" 또는 "좌/우 사진 추가 촬영" 선택
   ↓
3. 분석 시작
```

#### 옵션 B: 한 번에 선택
```
1. 정면 사진 촬영/선택
   ↓
2. "좌/우 사진도 촬영하시겠어요?" (선택사항)
   - 예: 좌/우 촬영 후 분석
   - 아니오: 정면만으로 분석 시작
```

### 3. 구현 계획

1. **CameraCapture 컴포넌트**: 정면만 촬영 (기본)
2. **선택적 MultiAngle**: "더 정확한 분석" 버튼으로 좌/우 추가
3. **분석 시작**: 정면만 있어도 즉시 시작 가능

## 📊 기대 효과

1. **사용자 편의성 향상**: 정면만으로 빠른 분석
2. **이탈률 감소**: 간단한 프로세스
3. **선택권 제공**: 원하는 사용자는 좌/우 추가 가능
4. **분석 정확도**: 정면만으로도 충분, 좌/우는 보너스

## 💡 결론

**정면만 필수, 좌/우 선택사항**으로 변경하는 것이 더 나은 UX입니다.
